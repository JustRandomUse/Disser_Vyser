# Анализ текущего состояния AirMonitorSystem

## Дата анализа: 2026-04-22

## 1. Текущая реализация

### Calendar.vue
✅ **Реализовано:**
- Режимы выбора: day/month/year
- Кнопка "Сегодня" работает мгновенно
- Выбор диапазонов дней/месяцев/годов
- Преобразование месяцев/годов в диапазоны дат
- Русификация

❌ **Проблемы:**
- Нет явного переключателя режимов в UI (есть в коде, но не видно пользователю)
- Визуальное оформление может быть улучшено

### Timeline.vue
✅ **Реализовано:**
- Отображение точек по дням/месяцам/годам
- Формат отображения даты/диапазона
- Выбор диапазона через Shift+клик
- Визуальная индикация выбранного диапазона
- Emit события 'range-selected'

✅ **Формат точек унифицирован:**
```javascript
{
  type: 'hour' | 'day' | 'month' | 'year',
  time: 'отображаемый текст',
  color: '#5DADE2',
  startDate: Date,
  endDate: Date,
  hour?: number
}
```

### App.vue
✅ **Реализовано:**
- generateTimePointsForRange() с автоматическим определением масштаба
- onTimeSelected() обрабатывает разные типы точек
- onRangeSelected() загружает временные ряды
- Передача данных в StatisticsModal

❌ **Проблемы:**
- Нет явного состояния для режима выбора (single-point / range)
- Карта не обновляется средними значениями при выборе диапазона

### StatisticsModal.vue
✅ **Реализовано:**
- Поддержка временных рядов (timeSeriesData, dateRange, rangeType)
- Линейный график с markLine (среднее/мин/макс)
- Candlestick график для мгновенных значений
- Таблица статистики

❌ **Проблемы:**
- График показывает только один параметр (pm25), нет выбора
- Нет переключения между параметрами

### api.js
✅ **Реализовано:**
- fetchAirQualityData() - актуальные данные
- fetchAggregatedData() - агрегированные данные
- fetchAverageData() - средние значения за период
- fetchTimeSeriesData() - временные ряды

✅ **Backend API поддерживает:**
- `/api/datasets/:code/last` - актуальные данные
- `/api/datasets/:code/data?date=...&hour=...` - данные за час
- `/api/datasets/:code/aggregated?time_begin=...&time_end=...&interval=...` - агрегированные

### MapView.vue
❌ **Проблемы:**
- При клике на станцию показывает только текущие данные
- Не учитывает выбранный временной диапазон
- Не показывает средние значения за период

### SidePanel.vue
❌ **Проблемы:**
- Нет кнопки "Показать средние по району за период"
- Не интегрирован с выбором временного диапазона

## 2. Что нужно доработать

### Приоритет 1: Критичные доработки

#### 1.1 MapView - показ средних значений
**Проблема:** Карта всегда показывает текущие данные, не учитывает выбранный период.

**Решение:**
- Добавить props: `mode` ('instant' | 'range'), `dateRange`
- При mode='range' показывать средние значения за период
- Обновлять цвета маркеров на основе средних значений

#### 1.2 App.vue - синхронизация состояния
**Проблема:** Нет единого состояния для режима работы.

**Решение:**
- Добавить `selectionMode` ref: 'instant' | 'range'
- При выборе диапазона в Timeline переключать режим
- Передавать режим в MapView и другие компоненты

#### 1.3 StatisticsModal - выбор параметра
**Проблема:** График показывает только PM2.5.

**Решение:**
- Добавить UI для выбора параметра (pm25, pm10, temperature, humidity, pressure)
- Перестраивать график при смене параметра

### Приоритет 2: Улучшения UX

#### 2.1 Calendar - явный переключатель режимов
**Решение:**
- Сделать переключатель режимов более заметным
- Добавить иконки для режимов

#### 2.2 Timeline - улучшение выбора диапазона
**Решение:**
- Добавить подсказку "Shift+клик для выбора диапазона"
- Показывать выбранный диапазон в человекочитаемом формате

#### 2.3 SidePanel - средние по району
**Решение:**
- Добавить кнопку "Показать статистику по району"
- При клике загружать средние значения по всем станциям района

### Приоритет 3: Полировка

✅ Уже реализовано через dateFormat.js

#### 3.2 Графики
