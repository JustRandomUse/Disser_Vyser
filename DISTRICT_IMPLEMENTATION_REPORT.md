# Реализация геометрической привязки районов

## Выполненные изменения

### 1. Установлены зависимости
```bash
npm install @turf/boolean-point-in-polygon @turf/helpers
```

### 2. Созданы файлы

#### `front/src/data/krasnoyarsk-districts.geojson`
GeoJSON файл с границами 7 административных районов Красноярска:
- Железнодорожный район
- Кировский район
- Ленинский район
- Октябрьский район
- Советский район
- Свердловский район
- Центральный район

**Источник данных:** Границы построены на основе анализа координат станций из API и общедоступной информации об административном делении Красноярска.

**Формат:** GeoJSON FeatureCollection, CRS: EPSG:4326 (WGS84)

**Лицензия:** Данные основаны на публичной административной информации. Для production рекомендуется получить официальные границы из:
- OpenStreetMap (когда доступны корректные relation ID)
- Администрация города Красноярска
- Росреестр (кадастровые данные)

#### `front/src/utils/districtMapping.js`
Утилита для геопривязки станций к районам:

**Функции:**
- `getDistrictForSensor(lon, lat)` - определяет район по координатам
- `groupSensorsByDistrict(sensors)` - группирует станции по районам
- `getAvailableDistricts()` - возвращает список всех районов
- `validateDistrictMapping(sensors)` - валидирует привязку и выводит отчет

**Технология:** @turf/boolean-point-in-polygon для point-in-polygon проверки

   - [ ] Выбрать "Кировский район"

   - [ ] Выбрать район в SidePanel
   - [ ] Убедиться, что выделенные станции географически находятся в этом районе
   - [ ] Проверить 2-3 станции визуально по карте

5. **Проверить консоль**
   - [ ] Открыть DevTools → Console
   - [ ] Найти "District mapping validation"

## Известные ограничения

### 1. Упрощенные границы районов
**Проблема:** GeoJSON содержит упрощенные прямоугольные полигоны, а не реальные административные границы.

**Причина:** OpenStreetMap API недоступны во время реализации.

**Решение:** Границы построены на основе анализа координат станций и общедоступной информации. Для production необходимо:
- Получить официальные границы из Росреестра
- Или использовать детальные данные OpenStreetMap (когда доступны)
- Или запросить у администрации города Красноярска

### 2. Станции на границах
**Проблема:** Некоторые станции могут находиться на границах районов.

**Текущее поведение:** Станция попадает в первый подходящий район (порядок проверки определяется порядком в GeoJSON).

**Рекомендация:** При получении точных границ проверить спорные станции вручную.

### 3. Новые станции
**Поведение:** Новые станции автоматически попадут в районы при следующей загрузке данных, если их координаты попадают в полигоны.

**Преимущество:** Не требуется изменение кода при добавлении новых станций в API.

## Data Quality / Follow-up

### Для улучшения точности:

1. **Получить официальные границы районов:**
   - Запросить у администрации Красноярска
   - Использовать данные Росреестра
   - Дождаться доступности OpenStreetMap Overpass API

2. **Валидация границ:**
   - Сравнить с официальными картами города
   - Проверить каждую станцию вручную по адресу
   - Уточнить спорные случаи на границах

3. **Расширение функциональности:**
   - Добавить визуализацию границ районов на карте
   - Показывать название района при наведении на станцию
   - Добавить фильтр по нескольким районам одновременно
