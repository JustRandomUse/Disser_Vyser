# Проверка геопривязки районов

## Как проверить работу

### 1. Запустить приложение

Backend:
```bash
cd back
go run cmd/main.go
```

Frontend (уже запущен на http://localhost:5175):
```bash
cd front
npm run dev
```

### 2. Открыть браузер

Открыть http://localhost:5175 и открыть DevTools (F12)

### 3. Проверить консоль

В консоли должно появиться сообщение:
```
District mapping validation: {
  total: 25,
  assigned: X,
  unassigned: Y,
  multipleDistricts: [],
  byDistrict: { ... }
}
```

### 4. Проверить SidePanel

1. Открыть боковую панель (кнопка справа)
2. В селекте "Районы города" должны быть:
   - Железнодорожный район
   - Кировский район
   - Ленинский район
   - Октябрьский район
   - Советский район
   - Свердловский район
   - Центральный район
   - Без района

3. Выбрать "Советский район" - должны выделиться станции на левом берегу
4. Выбрать "Кировский район" - должны выделиться станции на востоке
5. Выбрать "Центральный район" - должны выделиться станции в центре

### 5. Ручная проверка в консоли браузера

Вставить в консоль:
```javascript
// Импортировать функции (если доступны глобально)
// Или проверить через Vue DevTools

// Проверить конкретную станцию
const akadem = { longitude: 92.762222, latitude: 55.986571 };
// Должна быть в Советском районе

const kirovsky = { longitude: 92.958275, latitude: 55.994686 };
// Должна быть в Кировском районе
```

## Известные проблемы

1. **Упрощенные границы** - GeoJSON содержит прямоугольные полигоны, не точные административные границы
2. **Станции на границах** - могут попасть не в тот район (о. Молокова, о. Татышев)
3. **Требуется уточнение** - для production нужны официальные границы из Росреестра или OSM
